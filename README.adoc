= vertx-crate-client

A reactive CrateDB client for Eclipse Vert.x 5. It implements the Vert.x SQL Client API where applicable and adds CrateDB-specific APIs for low-level and streaming queries.

* Repository: https://github.com/NoEnv/vertx-crate-client[GitHub]
* License: link:http://www.apache.org/licenses/LICENSE-2.0.txt[Apache 2.0]

== Using vertx-crate-client

Add the following dependency to the _dependencies_ section of your build descriptor:

* Maven (in your `pom.xml`):

[source,xml,subs="+attributes"]
----
<dependency>
  <groupId>com.noenv</groupId>
  <artifactId>vertx-crate-client</artifactId>
  <version>5.0.9</version>
</dependency>
----

* Gradle (in your `build.gradle` file):

[source,groovy,subs="+attributes"]
----
compile 'com.noenv:vertx-crate-client:5.0.9'
----

== Quick start

Connect and run a SQL query:

[source,java]
----
CrateConnection.connect(vertx, new CrateConnectOptions())
  .compose(conn -> conn.query("SELECT 1 AS one").execute())
  .onSuccess(rows -> { /* use rows */ })
  .onFailure(Throwable::printStackTrace);
----

== Usage

*SQL query* (Vert.x SQL Client style, returns `RowSet<Row>`):

[source,java]
----
conn.query("SELECT id, name FROM doc.my_table LIMIT 10").execute()
  .onSuccess(rowSet -> { /* iterate rowSet */ });
----

*SQL with mapping or collector*:

[source,java]
----
conn.query("SELECT id FROM doc.t LIMIT 3").mapping(row -> row.getInteger("id")).execute();
conn.query("SELECT id FROM doc.t LIMIT 3")
  .collecting(Collectors.mapping(row -> row.getInteger("id"), Collectors.toList())).execute();
----

*Low-level request* (`CrateQuery` → `Future<CrateMessage>` with raw cols, types, rows as JSON):

[source,java]
----
conn.query(new CrateQuery("SELECT 1")).onSuccess(msg -> { /* msg.getRows(), msg.getCols(), ... */ });
// Parameterized:
conn.query(new CrateQuery("SELECT * FROM doc.t WHERE id = ?").setArgs(new JsonArray().add(42)))
  .onSuccess(msg -> { /* msg.getRows() */ });
// With options (default schema, column types, error trace):
CrateQueryOptions opts = new CrateQueryOptions().setDefaultSchema("doc");
conn.query(new CrateQuery("SELECT * FROM t", opts));
----

*Streaming* (`RowStream<JsonObject>`):

[source,java]
----
RowStream<JsonObject> stream = conn.streamQuery(new CrateQuery("SELECT * FROM doc.large_table"));
stream.handler(row -> { /* process row */ }).endHandler(v -> stream.close());
----

== About

A CrateDB client for Vert.x 5 with connection URI/programmatic config, SSL, failover, SQL and streaming APIs. See the link:src/main/asciidoc/index.adoc[full documentation] for connection options, failover, capabilities, and more.
